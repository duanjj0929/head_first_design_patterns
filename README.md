# Head First设计模式 - 学习笔记

## OO基础

- 抽象
- 封装
- 多态
- 继承

## OO原则

- 封装变化 - 找出应用中可能需要变化之处，把它们独立出来，不要和那些不需要变化的代码混在一起。
- 针对接口编程,不针对实现编程。
- 组合/聚合复用原则 - 多用组合,少用继承。
- 为交互对象之间的松耦合设计而努力。
- 开闭原则 - 类应该对扩展开放,对修改关闭。
- 依赖倒置原则 - 要依抽象,不要依赖具体类。
- 最少知识原则 - 只和你的密友谈话。
- 好莱坞原则 - 别调用(打电话给)我们，我们会调用(打电话给)你。
- 单一责任 - 一个类应该只有一个引起变化的原因。
